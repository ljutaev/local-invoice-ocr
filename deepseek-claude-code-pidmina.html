DeepSeek як бекенд Claude Code
підміна моделі, ціни на API та GitHub-проєкти

Технічний звіт · доповнення DeepSeek V4 · Claude Code Дата: 2026-06-05  ·  Джерела: офіційні доки DeepSeek API + GitHub-репозиторії (звірено вручну)

Цей файл — окреме доповнення до звіту про локальний LLM. Тема: чи можна «підмінити» дорогий хмарний Claude у Claude Code на дешевший DeepSeek, скільки це коштує і якими проєктами роблять. Ціни взято з офіційної сторінки DeepSeek [1], метод інтеграції — з офіційної доки [2]. Ціни змінюються — звіряйте перед оплатою.

1. Коротка суть (TL;DR)

Так, підміна працює — і офіційно
  • Claude Code звертається до Anthropic Messages API. DeepSeek надає Anthropic-сумісний ендпойнт https://api.deepseek.com/anthropic — тобто можна перенаправити Claude Code на DeepSeek двома змінними оточення, без жодного проксі [2][3].
  • Економія величезна: проєкти заявляють 17×–95× дешевше за підписку Anthropic (~$7–20/міс проти ~$200/міс) [5][6], переважно завдяки автоматичному кешуванню контексту в агентних циклах.
  • Платите ви за токени DeepSeek — це не «безкоштовно». «Free» у назвах проєктів = безкоштовний софт-проксі, а не безкоштовний inference.
  • Приватність: DeepSeek — хмарний провайдер (Китай). Дані виходять з вашого Mac на їхні сервери — протилежність локальному Ollama. Для конфіденційного коду це суттєво.

2. Як працює підміна бекенда

Claude Code дозволяє перевизначити, куди слати запити, через змінні оточення. Оскільки DeepSeek «розмовляє» тим самим протоколом, що й Anthropic, підміна — це просто інша адреса + ключ:

export ANTHROPIC_BASE_URL=https://api.deepseek.com/anthropic
export ANTHROPIC_AUTH_TOKEN=<ваш DeepSeek API-ключ>

Є три рівні «глибини» підміни:

ПідхідЩо робитьПроксі?
Офіційний (env-змінні)DeepSeek-ендпойнт напряму; одна модель або мапінг тірівНі
Локальний проксі-роутерПерехоплює запити, маршрутизує opus/sonnet/haiku на різні моделі/провайдериТак (локальний)
MCP-серверДодає DeepSeek як інструмент усередину Claude Code, не підмінюючи основну модель—

3. Офіційний метод DeepSeek + Claude Code

За офіційною документацією DeepSeek [2]. Передумови: Node.js 18+, встановлений Claude Code (npm install -g @anthropic-ai/claude-code), ключ з platform.deepseek.com/api_keys.

# macOS / Linux
export ANTHROPIC_BASE_URL=https://api.deepseek.com/anthropic
export ANTHROPIC_AUTH_TOKEN=<ваш DeepSeek API-ключ>
export ANTHROPIC_MODEL=deepseek-v4-pro
export ANTHROPIC_DEFAULT_OPUS_MODEL=deepseek-v4-pro
export ANTHROPIC_DEFAULT_SONNET_MODEL=deepseek-v4-pro
export ANTHROPIC_DEFAULT_HAIKU_MODEL=deepseek-v4-flash
export CLAUDE_CODE_SUBAGENT_MODEL=deepseek-v4-flash
export CLAUDE_CODE_EFFORT_LEVEL=max

claude

Як мапляться «тіри» Claude

Запит Claude Code→ Модель DeepSeek
claude-opus-*deepseek-v4-pro (потужніша)
claude-sonnet-* / claude-haiku-*deepseek-v4-flash (швидша/дешевша)

Тобто фонові й легкі задачі йдуть на дешевий Flash, а «важкі» (opus) — на Pro. Можна посадити все на Flash, щоб мінімізувати вартість.

Нюанс Web search у Claude Code генерує додаткові API-запити → додаткові витрати токенів [2]. А деякі сторонні гайди зазначають, що allowlist Claude Code не приймає сирі імена DeepSeek-моделей напряму — тоді запускають з --model sonnet/opus і прапорцем --bare, щоб обійти OAuth/keychain [6].

Застарілі (підтримка до 2026-07-24): deepseek-chat → v4-flash (non-thinking), deepseek-reasoner → v4-flash (thinking). DeepSeek залишає за собою право змінювати ціни — наведене не є офертою.

Чому це так дешево — кешування Контекстне кешування ввімкнено за замовчуванням. В агентних циклах системний промпт і контекст файлів повторюються, тож після першого запиту вони йдуть як cache hit — на Flash це $0.0028/M проти $0.14/M (≈98% економії). Саме тому реальні сесії коштують копійки.

6. GitHub-проєкти для підміни (зокрема «free-claude-code»)

Це проєкт, який ви згадували — клас «free claude code». Насправді їх кілька; ось ключові:

free-claude-code (Alishahryar1) [4]

Найбагатший за функціоналом проксі: маршрутизує Anthropic API-трафік Claude Code на 17 провайдерів — NVIDIA NIM, OpenRouter, Google AI Studio (Gemini), DeepSeek, Mistral, Kimi, Cerebras, Groq, Fireworks, Z.ai, LM Studio, llama.cpp, Ollama та ін. Підтримує per-model routing (Opus/Sonnet/Haiku/fallback на різні провайдери) і нативний /model-пікер. Працює і з CLI, і з VS Code / JetBrains, і з чат-ботами. Назв-двійників на GitHub кілька (Andrewkeith83 — через NVIDIA NIM + Telegram-бот; Rishurajgautam24) — звіряйте автора.

deepclaude (aattaran) [5]

Локальний проксі на localhost:3200; зберігає весь UX Claude Code (редагування файлів, bash, git). Бекенди: DeepSeek V4 Pro (дефолт), OpenRouter, Fireworks, Anthropic. Заявлено 17× дешевше (~$20/міс проти $200). Ліцензія MIT. Запуск через deepclaude.sh.

Обмеження deepclaude (важливо)
  • Vision/зображення через Anthropic-ендпойнт DeepSeek не підтримується
  • Паралельний виклик інструментів деградує
  • Немає підтримки MCP-серверних інструментів
  • Складне міркування інколи краще на Claude Opus

claudecode-deepseek-stack (MG-Cafe) [6]

Найближче до «чистої підміни двома env-змінними» — без стороннього проксі. Ключ кладуть у ~/.config/mg-deepseek/key.env, налаштування-оверрайд (що нейтралізує дефолти Vertex/Bedrock порожніми рядками) — у JSON, запуск: claude --bare --settings …. Заявлено ~95× дешевше (~$7/міс на Flash). Обмеження: треба --model sonnet/opus (allowlist не приймає імена DeepSeek), потрібен --bare.

Інші (для повноти)

  • DeepSeek-V4-Claude-Code-MCP — безкоштовний MCP-сервер, що додає DeepSeek як інструмент у Claude Code/Cursor/Codex (не підміняє основну модель, а доповнює; локально-first, 1M контекст).
  • yksanjo/deepseek-code — окремий open-source CLI на DeepSeek-V4 («свій Claude Code за 1/100 ціни») — це не підміна, а альтернативний клієнт.
  • raine/claude-code-proxy — проксі, щоб ганяти Claude Code через підписку ChatGPT/Kimi замість API [7].

7. Застереження та ризики

Приватність даних DeepSeek — хмарний сервіс. Увесь ваш код, контекст файлів і промпти йдуть на сервери DeepSeek (юрисдикція — Китай) і підлягають їхній політиці зберігання. Це протилежність локальному Ollama, де нічого не виходить з Mac. Для коду під NDA / персональних даних — серйозний стоп-фактор. Якщо приватність критична — лишайтесь на локальному Ollama (див. основний звіт), а DeepSeek-підміну використовуйте лише для некритичного коду.
  • Функціональні втрати через Anthropic-ендпойнт DeepSeek: немає vision, деградує паралельний tool use, немає MCP-інструментів [5]. Для агентних робочих процесів Claude Code це відчутно.
  • Якість. DeepSeek V4 сильний у кодингу, але це не паритет з фронтирним Claude на складному міркуванні/довгих агентних ланцюгах.
  • «Free» ≠ безкоштовно. Безкоштовний лише проксі-софт; за inference DeepSeek платите за токени (хоча й дешево).
  • ToS / стабільність. Підміна бекенда сторонніми env-змінними не порушує ToS Claude Code сама по собі, але це не офіційно підтримуваний провайдер (на відміну від Bedrock/Vertex), тож гарантій якості/сумісності немає; оновлення Claude Code можуть ламати обхідні прапорці типу --bare.
  • Безпека ключів. Не комітьте key.env / API-ключі в git.

8. Рекомендація

Як обрати
  • Хочете найпростіше й офіційно: env-метод DeepSeek (розділ 3) — дві основні змінні + мапінг тірів, без проксі.
  • Хочете гнучку маршрутизацію (різні моделі/провайдери на різні тіри, у т.ч. локальний Ollama як fallback): free-claude-code (Alishahryar1) або deepclaude.
  • Приватність понад усе: не DeepSeek, а локальний Ollama з основного звіту. DeepSeek-підміна — для економії на некритичних задачах.
  • Гібрид: Claude Code на DeepSeek-Flash для рутини + перемикання на справжній Claude для складного. Саме це й дають роутер-проксі.
